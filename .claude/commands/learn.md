# Learn - Extract Reusable Patterns

Analyze the current session and extract patterns worth saving.

## Instructions

Review the current session for:

1. **Error Resolution Patterns**
   - What error occurred?
   - What was the root cause?
   - What fixed it?

2. **Debugging Techniques**
   - Non-obvious debugging steps
   - Tool combinations that worked

3. **Cashe-Specific Patterns**
   - Codebase conventions discovered
   - Architecture decisions made
   - Supabase/cache patterns

4. **Workarounds**
   - Library quirks
   - Supabase limitations
   - Vite/React specific fixes

## Process

1. Review the session for extractable patterns
2. Identify the most valuable/reusable insight
3. Check existing memory files for duplicates
4. Update MEMORY.md or create a topic-specific file in the memory directory
5. Ask user to confirm before saving

## What NOT to save
- Trivial fixes (typos, syntax errors)
- One-time issues
- Session-specific context
- Anything already in CLAUDE.md
